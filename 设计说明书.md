## 1. 系统概述

### 1.1 产品定位

OD_FUSION 是一款面向量產車型的多傳感器融合障礙物感知系統。系統整合環視攝像頭（SVS）、鳥瞰視圖（BEV）和雷達（Radar）三大傳感器，實現對前方目標的穩定跟蹤與準確識別。

### 1.2 系統功能

| 功能 | 說明 |
|------|------|
| 多傳感器融合 | 支持SVS、BEV、Radar三種傳感器數據融合 |
| 目標跟蹤 | 對障礙物目標進行穩定的時序跟蹤 |
| 類型識別 | 識別目標類型（車輛、行人、騎行者等） |
| 運動狀態估計 | 估計目標的運動狀態（移動、静待、穿越等） |
| 輸出管理 | 輸出穩定的融合結果 |

### 1.3 性能指標

| 指標 | 數值 |
|------|------|
| 融合周期 | 50ms |
| 最大輸入目標數（SVS） | 64 |
| 最大輸入目標數（BEV） | 20 |
| 最大輸入目標數（Radar） | 64 |
| 最大輸出航跡數 | 64 |
| 航跡管理器深度 | 30 |
| 航跡管理器寬度 | 64 |

## 3. 坐標系定義

### 3.1 ISO8855 國際標準

車輛坐標系 ISO8855 定義：
- X軸：前進方向
- Y軸：左側方向
- Z軸：上方

### 3.2 傳感器坐標系

| 傳感器 | 原點位置 | X軸 | Y軸 |
|--------|---------|-----|-----|
| SVS | 後軸中心 | 前 | 左 |
| BEV | 後軸中心 | 前 | 左 |
| Radar | 前保險槓中心 | 前 | 左 |

### 3.3 坐標轉換

#### 笛卡爾坐標轉 ISO8855

```cpp
void CartesianToIso8855(float x, float y, float* iso_x, float* iso_y) {
  *iso_x = -y;
  *iso_y = x;
}
```

#### 局部坐標轉全局坐標

```cpp
void LocalToGlobal(const GlobalPose& pose, float local_x, float local_y,
                   float* global_x, float* global_y) {
  float cos_heading = std::cos(pose.heading);
  float sin_heading = std::sin(pose.heading);
  *global_x = pose.x + cos_heading * local_x - sin_heading * local_y;
  *global_y = pose.y + sin_heading * local_x + cos_heading * local_y;
}
```

## 5. 融合算法設計

### 5.1 卡爾曼濾波器（CV模型）

系統使用常速（Constant Velocity）卡爾曼濾波器進行狀態估計。

#### 狀態向量

```
X = [x, y, vx, vy]^T
```

#### 狀態轉換矩陣

```
     ┌             ┐
F =  │ 1  0  dt  0  │
     │ 0  1  0   dt │
     │ 0  0  1   0  │
     │ 0  0  0   1  │
     └             ┘
```

#### 過程噪聲

```cpp
// 位置過程噪聲：0.001
// 速度過程噪聲：0.02
```

#### 預測方程

```cpp
x_pred = F * x_est
P_pred = F * P_est * F^T + Q
```

#### 更新方程

```cpp
K = P_pred * H^T * (H * P_pred * H^T + R)^(-1)
x_est = x_pred + K * (z - H * x_pred)
P_est = (I - K * H) * P_pred
```

### 5.2 數據關聯

使用匈牙利算法進行二維分配問題求解。

#### 代價函數

```cpp
cost = sqrt(dist² * kPosWeight + vel_diff² * kVelWeight)
```

| 參數 | 數值 |
|------|------|
| kDistGate | 20.0f |
| kPosWeight | 1.0f |
| kVelWeight | 0.1f |
| kCostThreshold | 10.0f |

#### ID匹配

當觀測目標與歷史目標的匹配ID相同時，賦予極低成本（0.01）以確保ID正確的目標優先匹配。

### 5.3 傳感器特定處理

#### SVS處理

- 坐標系：ISO8855
- 有效範圍：X ∈ [-10, 10], Y ∈ [-8, 20]
- 坐標轉換：無需轉換

#### BEV處理

- 坐標系：ISO8855
- 特殊處理：x坐標加上veh_head_rear_wheel（軸距補償）
- 有效範圍：X ∈ [-10, 10], Y ∈ [-8, 20]

#### Radar處理

- 坐標系：笛卡爾坐標
- 坐標轉換：笛卡爾 → ISO8855
- 速度補償：vx = vx - veh_spd（去除自車速度）

## 6. 航跡管理

### 6.1 航跡類（Track）

每個航跡封裝完整的跟蹤狀態：

```cpp
class Track {
  int32_t track_id_;              // 航跡ID
  uint8_t cursor_;                // 循環緩冲區指針
  TrackStatus status_;            // 航跡狀態
  CvKalmanFilter kalman_filter_; // 卡爾曼濾波器
  std::vector<FusedObject> matrix_;   // 歷史觀測 (kTrackDepth)
  std::vector<uint8_t> lst_flg_;      // 歷史標誌 (kTrackDepth)
  FusedObject estimated_;         // 當前估計
  FusedObject output_;            // 輸出
  uint8_t meas_flag_;             // 測量標誌
};
```

### 6.2 航跡狀態機

```
                    ┌───────────────┐
                    │  kUntracking  │
                    └───────────────┘
                            │
                            ▼
                    ┌───────────────┐
         ┌──────────│   kInitial    │◄─────────┐
         │          └───────────────┘          │
         │                 │                   │
         │                 ▼                   │
         │          ┌───────────────┐         │
         │          │   kGrowing     │─────────┘
         │          └───────────────┘
         │                 │
         │                 ▼
         │          ┌───────────────┐         │
         │          │   kStable      │─────────┘
         │          └───────────────┘
         │                 │
         │                 ▼
         │          ┌───────────────┐
         └─────────►│   kUnstable    │
                    └───────────────┘
```

### 6.3 狀態轉換條件

| 當前狀態 | 條件 | 下一狀態 |
|---------|------|---------|
| kUntracking | 創建新航跡 | kInitial |
| kInitial | cnt < 3 | kInitial |
| kInitial | cnt >= 3 | kGrowing |
| kGrowing | cnt >= 3 | kStable |
| kStable | lst >= 2 | kUnstable |
| kUnstable | lst < 2 | kStable |
| any | cnt = 0 | kUntracking |

### 6.4 計數器管理

每個航跡維護多個計數器：

```cpp
struct TrackStatus {
  int32_t cnt = 0;           // 總命中計數
  int32_t lst = 0;          // 連續丟失計數
  int32_t cnt_svs = 0;      // SVS命中計數
  int32_t lst_svs = 0;      // SVS連續丟失
  int32_t cnt_bev = 0;      // BEV命中計數
  int32_t lst_bev = 0;      // BEV連續丟失
  int32_t cnt_radar = 0;    // Radar命中計數
  int32_t lst_radar = 0;    // Radar連續丟失
  int32_t cnt_fused = 0;    // 融合命中計數
};
```

### 6.5 航跡生命周期

```
創建 ──► Initial ──► Growing ──► Stable
                         │            │
                         ▼            ▼
                      Unstable ◄─────┘
                         │
                         ▼
                     刪除/替換
```

## 7. 傳感器接口

### 7.1 OdFusionComponent 配置

```cpp
struct OdFusionComponent::Config {
  bool enable_svs = true;    // 啟用SVS融合
  bool enable_bev = false;  // 啟用BEV融合
  bool enable_radar = false;// 啟用Radar融合
};
```

### 7.2 數據轉換函數

#### SVS轉換

```cpp
std::vector<FusedObject> ConvertSvsToObservations(const SvsFrame& svs_frame);
```

轉換規則：
- 直接使用原始坐標
- obj_det_prop = kSoleSvs
- svs_match_id = object.id + 1

#### BEV轉換

```cpp
std::vector<FusedObject> ConvertBevToObservations(
    const BevFrame& bev_frame, float veh_head_rear_wheel);
```

轉換規則：
- x坐標加上veh_head_rear_wheel補償
- obj_det_prop = kSoleBev
- bev_match_id = object.id + 1

#### Radar轉換

```cpp
std::vector<FusedObject> ConvertRadarToObservations(
    const RadarFrame& radar_frame, float veh_spd);
```

轉換規則：
- 坐標：笛卡爾 → ISO8855
- 速度：vx = vx - veh_spd
- obj_det_prop = kSoleRadar
- radar_match_id = object.id + 1

### 7.3 統一處理接口

```cpp
void OdFusionComponent::Process(const FrameData& frame_data, uint64_t meas_time);
```

內部自動根據配置調用對應傳感器處理函數。
